Industrial Fault Detection System

PyTorch · LSTM · Isolation Forest · FastAPI · Docker

GitHub Repository

Problem

Industrial machinery failures cause significant downtime and safety risks. Traditional threshold-based monitoring systems produce high false positive rates and miss subtle degradation patterns. The challenge: detect anomalies in 6-axis IMU sensor data (accelerometer + gyroscope) with high precision in real-time industrial environments.

Technical Approach

Architecture

Ensemble system combining two complementary approaches:

  • LSTM Autoencoder with Attention: Captures temporal dependencies in sensor sequences. Attention mechanism identifies which time steps contribute most to reconstruction error.
  • Isolation Forest: Handles non-sequential anomalies and provides interpretable feature importance scores.

Data Pipeline

  • Sliding window approach (100-sample windows, 50-sample overlap)
  • Z-score normalization per sensor axis
  • Feature engineering: statistical moments, FFT features, peak detection

Model Details

LSTM Autoencoder:

  • Encoder: 2-layer LSTM (128→64 hidden units)
  • Bottleneck: 32-dimensional latent space
  • Decoder: 2-layer LSTM with attention over encoder hidden states
  • Loss: MSE reconstruction + KL divergence regularization

Isolation Forest:

  • 100 trees, contamination=0.1
  • Applied to concatenated statistical features

Results

  • Precision: 94.2% on held-out test set
  • Recall: 89.7%
  • False Positive Rate: 2.1%
  • Inference Time: 12ms per window (CPU)

Ensemble approach outperformed individual models by 7-9% on F1 score. Attention mechanism successfully identified bearing wear signatures 3-5 days before critical failure.

Implementation

Production Deployment

  • FastAPI service: REST endpoints for real-time inference and batch processing
  • Model versioning: MLflow integration for experiment tracking
  • Docker containerization: Reproducible deployment across environments
  • Logging: Structured logging with anomaly scores and feature attributions

Key Challenges

  • Class imbalance: Solved with SMOTE oversampling + weighted loss
  • Temporal drift: Implemented sliding window retraining every 30 days
  • Real-time constraints: Optimized with TorchScript compilation

Learnings & Future Work

  • Ensemble methods significantly improve robustness in noisy industrial settings
  • Attention mechanisms provide interpretability critical for operator trust
  • Next steps: Transformer-based architecture, multi-sensor fusion, causal analysis for root cause identification
